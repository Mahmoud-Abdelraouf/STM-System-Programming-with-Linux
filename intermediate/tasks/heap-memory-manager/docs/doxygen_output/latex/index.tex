\hypertarget{index_autotoc_md31}{}\doxysection{Introduction}\label{index_autotoc_md31}
Memory management is a critical aspect of system programming, particularly in environments where resource efficiency and performance are paramount. This project presents a custom Heap Memory Manager (HMM) designed specifically for Linux systems, offering an alternative to the standard {\ttfamily malloc} and {\ttfamily free} functions with more efficient, flexible, and robust memory management capabilities.\hypertarget{index_autotoc_md32}{}\doxysection{Linux Memory Management Overview}\label{index_autotoc_md32}
\hypertarget{index_autotoc_md33}{}\doxysubsection{Why Custom Memory Management?}\label{index_autotoc_md33}
In typical Linux applications, memory management is handled by the system\textquotesingle{}s allocator, which uses functions like {\ttfamily malloc}, {\ttfamily calloc}, and {\ttfamily free}. While these standard functions are sufficient for many applications, they may not provide the level of control, efficiency, or customization required for certain high-\/performance or specialized applications. This project addresses these limitations by offering\+:


\begin{DoxyItemize}
\item {\bfseries{Improved Performance\+:}} Custom memory allocation algorithms can reduce fragmentation and enhance performance.
\item {\bfseries{Thread Safety\+:}} The GLib Thread (GLThread) module provides a thread-\/safe linked list implementation, essential for concurrent applications.
\item {\bfseries{Memory Usage Optimization\+:}} The HMM system is designed to make better use of available memory, reducing wastage and improving overall application efficiency.
\end{DoxyItemize}\hypertarget{index_autotoc_md34}{}\doxysection{Project Features}\label{index_autotoc_md34}

\begin{DoxyItemize}
\item {\bfseries{Heap Memory Manager (HMM)\+:}} A robust memory management system that replaces the standard dynamic memory allocation functions with custom, optimized alternatives.
\item {\bfseries{GLib Thread (GLThread)\+:}} A generic, thread-\/safe linked list implementation built using GLib, facilitating safe and efficient multi-\/threaded operations.
\item {\bfseries{Memory Manager Test Suite\+:}} A comprehensive test suite that rigorously tests the memory manager\textquotesingle{}s functionality and performance across various scenarios, ensuring reliability and robustness.
\item {\bfseries{Support for Static and Shared Libraries\+:}} The project can be built as either a static library ({\ttfamily .a}) or a shared library ({\ttfamily .so}), providing flexibility in how it can be integrated into different applications.

\item {\bfseries{Build Everything\+:}}

\begin{DoxyCode}{0}

\item {\bfseries{Build Static Library Only\+:}}

\begin{DoxyCode}{0}

This builds only the static library ({\ttfamily libhmm.\+a}).
